# Template Starter Contract

Starter blocks are optional accelerators layered on top of runtime blocks.

## Starter Blocks

- `admin-dashboard-starter`
- `mobile-crud-starter`

## Starter Intent

Starter blocks provide ready-made app structure for common workflows:

- dashboard home
- CRUD workspace
- list/detail/edit flows
- reusable shell components for admin-style products

## Starter Rules

- starter blocks are opt-in only
- starter blocks do not replace the approved product design
- starter blocks must still obey entity names, payload shapes, auth boundaries, and CRUD rules
- starter blocks can create a base shape, but the builders must still fill in real entity logic

## When To Use

Use starter blocks when the user asks for:

- a ready-made base
- bootstrap completo
- base pronta
- clone-style initial structure

Do not use starter blocks automatically for every app. Default to `contract-only` unless the user or architecture explicitly chooses `embedded-template`.
